===== 旧計算式アーカイブ (修正前) =====
記録日: 2026-06-11

【入力欄】
- 純資産合計 (netTotal)
- うち現物株評価額 (stock)
- ポジション総額・建値 (pos)
- 信用含み損益 (creditGain) ※評価損益のみ

【中間値】
現金     = 純資産 − 現物×0.8 − 信用含み損益
           ※問題: 掛目0.8を現金逆算に使用 → 現物×0.2 分が二重計上
           ※問題: 含み損をマイナス入力するため +|含み損| が現金に加算されてしまう
信用建玉 = ポジション − 現物 + (信用含み損益 < 0 ? |信用含み損益| : 0)
           ※問題: 分母が時価ベースになっていた

【現状サマリー維持率】
維持率 = (現金 + 現物×0.8) / 信用建玉 × 100

【シナリオ維持率 (dropPct = ポジション下落率)】
stockLoss  = 現物 × dropPct
creditLoss = 信用建玉 × dropPct
numer = 現金 + 現物×0.8 − 現物損×0.8 − (信用建玉損 − 信用含み損益)
分母  = 信用建玉 (固定)

【維持率20%逆算】
drop = (現金 + 現物×0.8 + 信用含み損益 − 0.20×信用建玉) / (現物×0.8 + 信用建玉)

【判明した誤り3点】
誤り1: 現金逆算で掛目0.8を使用 → 578×0.2=115.6万が二重計上
        さらに含み損をマイナス入力するため +14万が現金に誤加算
誤り2: 分母が建値でなく時価ベース (ポジション−現物 = 時価差額)
        正しくは建代金合計 (SBIの「建代金合計」欄) を使うべき
誤り3: 信用損益が評価損益のみ。決済損益・経費を含めるべき

【検証例 (SBI実績)】
入力: 純資産717, 現物578, ポジション1304, 信用含み損益-14
旧ツール結果: 100.7% (SBI実績88.73%より約12pt過大)
正しい計算: (224.4 + 578.8×0.8 − 31.4) / 739.3 = 88.7%

===== 第2次アーカイブ (建代金入力を追加した版) =====
記録日: 2026-06-11

【入力欄】
- 純資産合計 (netTotal)
- うち現物株評価額 (stock)
- ポジション総額・建値 (pos)
- 建代金合計 (costBasis) ← SBI「建代金合計(B)」
- 信用損益合計 (creditGain) ← 評価損益+決済損益+経費

【採用した計算式】
現金     = 純資産 − 現物（掛目なし）
維持率   = (現金 + 現物×0.8 + 信用損益合計) ÷ 建代金合計

【問題】
SBIの参考委託保証金率は (実質保証金A + スィープ口座C) / 建代金B であり、
ツールの「現金」はSBIの「委託保証金現金 + 代用有価証券評価額(掛目済み)」とは異なる。
純資産合計の定義とSBIの内訳が一致しなかったため、正確な再現が困難だった。

===== 正しいSBI維持率計算 =====
SBIの参考委託保証金率 = (A + C) / B × 100
  A = 実質保証金    (SBI「実質保証金(A)」欄から直接読み取り)
  C = スィープ口座  (SBI「スィープ専用銀行口座残高(C)」欄から直接読み取り)
  B = 建代金合計    (SBI「建代金合計(B)」欄から直接読み取り)

検証: (431.58 + 224.43) / 739.28 = 88.73% ✓

→ 入力3項目のみで正確に計算可能。逆算不要。
